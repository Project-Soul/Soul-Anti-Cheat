Settings

You can find the Settings module in the loader folder.

Whitelist#

The whitelist array must contain usernames or user ids. It decides whether or not a player should bypass the anti-cheat checks.

Example
Settings.Whitelist = {    "proxilom", --[ Username ]    894920299   --[ User Id ]}

ChecksSettings#

Holds a reference to Checks' settings.

Format
["CheckName"] = {    Disabled: boolean}
Default Settings
{    Disabled = false}
Example
Settings.ChecksSettings = {    ["Velocity"] = { Disabled = false }}

Reasons#

Reasons the player will be kicked for if they trigger a check. The index can be used with Global:Kick. It can hold curly brackets to hold extra data.

Example
Settings.Reasons = {    ["Velocity"] = "\nHigh Velocity\nVelocity: {Velocity}\n"}
-- Check:_G.SoulAntiCheat:Kick(Player, "Velocity", RootVelocity)

Physics#

Anything regarding physics.

MaxVelocity#

The greatest AssemblyLinearVelocity a player can have before getting kicked.

MaxStudsTeleport#

The greatest amount of studs a player can travel before getting kicked.

RaycastSettings#

Anything regarding raycasting.

MaxFlyDistance#

The greatest amount of studs away a player can be from a surface before getting kicked.

HumanoidSettings#

Anything regarding the Humanoid.

MaxConsecutiveJumps#

  • Deprecated

The greatest amount of jumps in the air a player can perform before getting kicked. (Do not use as this won't work)

RootSettings#

Anything regarding the HumanoidRootPart etc.

WaitForRoot#

Whether or not the anti-cheat needs to wait for the HumanoidRootPart.

ImportantParts#

If an Instance has it's name in this array, it will kick the player.

WebhookSettings#

Anything regarding Discord webhooks.

Enabled#

Whether or not logging should be enabled.

WebhookId#

Your webhook's id.

WebhookToken#

Your webhook's token.

Misc#

Anything miscellaneous.

VelocityDecimals#

The amount of decimals to be displayed in the Velocity check kick message.

TeleportBack#

Whether or not the Teleport check should teleport players back.
